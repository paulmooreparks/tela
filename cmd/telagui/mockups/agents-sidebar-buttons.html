What changed

Buttons drop the words Agent and Code so they fit side-by-side at the live 240px sidebar width: Generate… and Redeem…. The caption picks up the explanatory work, naming both pair code and the another machine vs here contrast that the labels used to carry. Tooltips on each button (not visible in the static mockup) keep the long form: "Generate a pairing code on this hub for an agent on a remote machine" and "Redeem a pair code to register a local agent on this machine."

Why the caption is fine doing the work

The caption is one sentence and sits directly under the buttons, so an operator scanning the sidebar for the first time sees the full explanation alongside the controls. After they've used either button once, the short labels are enough on their own. This is the same pattern as Undo / Save in every toolbar in TV: the label alone is meaningless out of context, but the surrounding pane makes it obvious.
