Agent pairing code flow (proposed)

Replaces the current "Pair Agent" button on the Agents tab. The button name and the result modal are the changes; the underlying code-generation API call is unchanged.

Step 1 — Agents tab sidebar
Button labeled for what it actually does
Agents
work-laptop
dev-vm
owlsnest-agent
Onboards a new agent on a remote machine

Old label: Pair Agent. Suggested a complete in-app handshake; really it just generates a code the operator copies to the agent machine.

New label: Generate Agent Code…. The ellipsis matches TDL's convention for buttons that open a dialog. The footer caption explains the purpose so an operator seeing this for the first time understands why this lives on the Agents tab and not somewhere else.

Step 2 — generation modal
Asks for machine name + expiration, in a real dialog

Generate Agent Code

Generate a one-time code that the operator on the new machine can exchange for an agent token using telad pair.

The name the agent will register under. Operators usually match the host's name. Required.

Replaces the current single-line prompt that only asked for the machine name and used the generic dialog primitive. A proper modal lets us add the expiration dropdown the user already sees on the Pair Code modal in the Access tab, plus room for a hint that explains what "machine name" actually means when this dialog opens for the first time.

Step 3 — result modal
Code + ready-to-run command, both with copy buttons

Agent Code Generated

Run the command below on the machine where you want to install the agent. The code can only be used once and expires shortly.

Code ABCD-1234
Command telad pair -hub wss://owlsnest.parkscomputing.com -code ABCD-1234
Machine: staging-runner Expires: 2026-04-22T05:30:00Z (in 1 hour)

After the command runs successfully on the agent machine, the agent will appear in the sidebar on the left. If the agent is offline or unreachable, the code stays valid until the expiration time above.

Today the result is shown in a generic prompt dialog that displays just the code. The new modal upgrades that with:

  • The code in large monospace, easy to read aloud or screenshot.
  • A pre-filled telad pair command line, so the operator can paste it straight into a chat or SSH session without hand-assembling it.
  • One Copy button per item.
  • Meta row with machine name and absolute expiry timestamp (operators have asked for absolute timestamps elsewhere).
  • A hint clarifying what happens after the operator runs the command, since pairing completion is the part the operator can't see in TV until it lands.
